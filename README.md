# 🤖 Butler AI v3: 자가 치유형 에이전트 시스템 가이드

본 문서는 갤럭시 S9(Termux) 환경에서 구동되는 **Butler AI v3** 시스템의 아키텍처, 기능 및 사용법을 정리한 공식 가이드입니다.

---

## 🏗️ 시스템 아키텍처 (Layered Architecture)

시스템은 실행부와 관리부가 분리되어 상호작용하는 구조로 설계되었습니다.

1. **Execution Layer (현장 요원 - `butler`)**
   - 파일: `butler_pro.py`
   - 역할: Discord 통신 인터페이스, 뉴스 수집, 기차 예약, Flask 서버 운영.
2. **Agent Layer (두뇌/관리 요원 - `agent`)**
   - 파일: `butler_agent.py`, `core/agent_manager.py`
   - 역할: 시스템 메트릭(온도/배터리) 감시, 에러 로그 분석, 자가 치유(Patch) 제안.
3. **Infrastructure Layer (인프라)**
   - 도구: **PM2**, **Python venv**
   - 역할: 24/7 무중단 가동, 프로세스 감시, 로그 로테이션.

## 🚀 주요 기능

### 1. 🏥 자가 치유 (Self-Healing)
- **에러 로그 감시**: `butler` 서비스에서 발생하는 모든 Exception을 실시간 감지.
- **Gemini 분석**: 에러 발생 시 Gemini AI가 로그를 분석하여 원인을 진단하고 해결 코드(Patch)를 생성.
- **승인 기반 패치**: 에이전트가 코드를 직접 수정하기 전 사용자에게 승인(`!승인`)을 요청하여 안전성 확보.

### 2. 📱 하드웨어 및 발열 관리
- **온도 모니터링**: S9 배터리 온도를 체크하여 45°C 이상 시 자동 경고 및 작업 지연(Throttling).
- **시스템 상태 보고**: 배터리 잔량, 전압, 건강 상태 등을 Discord Embed 형식으로 실시간 보고.

### 3. 🚆 스마트 기차 예약 (SRT/KTX)
- **예약 메뉴**: `!srt` 명령어를 통한 인터랙티브 예약 UI 제공.
- **예약 대기열**: 백그라운드 루프가 예약 가능 여부를 지속적으로 체크하여 자동 예약 시도.

### 4. 📰 지능형 뉴스 트래킹
- **키워드 관리**: `!뉴스 추가 [키워드]`를 통해 동적으로 수집 대상 변경 가능.
- **중복 제거**: 이미 보고된 뉴스는 캐싱하여 중복 알림 방지.

### 5. 🛠️ A2A (Agent-to-Agent) 자율 작업
- **Manager-Coder 협업**: 사용자의 요구사항을 분석하는 Manager와 코드를 짜는 Coder 에이전트가 협업하여 새로운 기능 구현.

## 💬 주요 명령어 (Discord)

### 🏥 관리 & 유지보수 (Chat 채널)
- `!승인 [ActionID]`: 에이전트가 제안한 자가 치유 작업을 승인 및 실행.
- `!모델 리스트`: 현재 설정된 AI 모델 확인 및 추천 목록 보기.
- `!모델 설정 [ModelID]`: 일반 대화용 Gemini 모델 변경.

### 🚆 기차 예약 (SRT 채널)
- `!srt`: SRT 예약 메인 메뉴 표시.
- `!역 리스트`: 현재 설정된 예약 가능 역 목록.
- `!역 추가 [역명]`: 새로운 SRT 정차역 추가.

### 📰 뉴스 알림 (News 채널)
- `!뉴스 리스트`: 현재 추적 중인 키워드 목록.
- `!뉴스 추가 [키워드]`: 새로운 뉴스 키워드 등록.

### 📱 시스템 상태 (Status 채널)
- "배터리 어때?", "온도 알려줘": 하드웨어 상태 실시간 보고.
- `!help`: 채널별 도움말 확인.

## 🛠️ 개발자/관리자 가이드 (Termux CLI)

### 서비스 관리 (PM2)
```bash
# 전체 상태 확인
pm2 list

# 실시간 모니터링 (CPU/메모리)
pm2 monit

# 특정 서비스 재시작
pm2 restart butler
pm2 restart agent

# 로그 확인
pm2 logs butler --lines 50
```

### 경로 정보
- **프로젝트 루트**: `~/dev_pjt/my_butler/`
- **로그 저장소**: `~/.pm2/logs/`
- **설정 파일**: `core/agent_state/agent_config.json`

## ⚠️ 주의사항
- **보안**: `.env` 파일의 API 키와 토큰이 노출되지 않도록 주의하십시오.
- **하드웨어**: S9은 배터리 팽창(스웰링) 위험이 있으므로 에이전트의 온도 경고 발생 시 즉시 점검이 필요합니다.
- **A2A**: 자율 코딩 작업 결과물은 실행 전 반드시 에이전트의 검증 보고서를 확인하십시오.
