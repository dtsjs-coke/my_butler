# LIMIT VWAP Trading Bot 사용자 가이드 (User Manual)

본 문서는 LIMIT VWAP 자동 매매 시스템의 핵심 로직, 매매 파라미터의 상세 정의, 그리고 실전 투자 고수들이 활용하는 대표적인 설정값 가이드를 설명합니다.

---

## 1. LIMIT VWAP 전략 핵심 요약

본 자동 매매 봇은 당일 리셋 시각부터 누적으로 **VWAP(거래량 가중 평균 가격)**을 실시간 연산하고, 이 VWAP 라인을 기준선 삼아 철저히 **지정가(Limit) 주문**을 제출하고 제어합니다.

### 📈 매매 규칙 개요
1. **진입 (지정가 매수)**:
   * 현재가가 VWAP 선보다 아래에 위치하고 보유 주식이 없을 때 작동합니다.
   * 실시간으로 **매수 타겟 가격**($TargetBuy = VWAP \times (1 - N\%)$)을 계산하여 **지정가 매수** 주문을 제출합니다. VWAP 선의 변동에 맞춰 미체결 매수 주문은 취소 및 재접수(Cancel & Replace)를 반복하며 실시간 추종합니다.
2. **청산 (지정가 매도)**:
   * 주식을 보유하고 있을 때 작동합니다.
   * 현재가가 VWAP 선을 다시 상향 돌파하거나, **매도 타겟 가격**($TargetSell = VWAP \times (1 + M\%)$) 이상 상승 시 **지정가 매도** 주문이 집행되거나 청산됩니다.
3. **손절 (Stop Loss)**:
   * 주식 평단가 대비 가격이 $-X\%$ 이하로 하락 시, 즉시 미체결 매도 주문을 취소하고 **시장가(Market)로 전량 즉시 청산**하여 추가 손실을 방지합니다.

## 2. 다중 가상 자동매매 및 종합 잔고

가상 자동매매의 경우 최대 3개의 개별 봇 인스턴스(**VIRTUAL_1, VIRTUAL_2, VIRTUAL_3**)를 동시에 구동할 수 있습니다.

* **가상거래 종합 잔고 현황**: 3개 가상 봇의 가용 현금과 주식 평가금을 합산하여 전체 총자산 및 누적 수익률(ROI), 합산 승률을 실시간 요약 표기합니다.
* **서브 탭 스위칭**: 대시보드의 `🤖 가상 봇 1` ~ `🤖 가상 봇 3` 단추를 누르면, 설정 화면, 차트 위젯, 주문 현황, 체결 이력, 터미널 로그가 해당 가상 봇의 정보로 즉시 전환됩니다.
* **실거래 듀얼 수익률 표기**: 실제 거래 연동 시 대시보드는 **TOSS 계좌 전체** 현황과 **VWAP 봇 전용** 성과를 듀얼 모드로 동시에 제공합니다.
  * *TOSS 계좌 전체*: 연동된 토스증권 계좌 내의 모든 주식 평가금과 예수금을 합산하여 '기준 자본금' 대비 ROI를 보여줍니다.
  * *VWAP 봇 전용*: 봇의 실제 주문 체결 이력(`vwap_trades_real.json`)의 누적 실현 손익과 봇이 가동 중인 종목의 평가 손익만을 취합하여 '기준 자본금' 대비 ROI를 독립적으로 산출합니다. 계좌 내 타 종목이나 수동 거래 건은 봇 수익률에서 자동 제외되어 알고리즘 자체의 객관적인 성과를 측정하기에 매우 적합합니다.

## 3. 매매 파라미터 상세 정의 및 실전 권장 대표값

실전에서 고수들이 변동성 및 자금 관리를 위해 사용하는 파라미터별 정의 및 대표 권장값 가이드입니다.

| 파라미터명 | 설명 | 고수들의 대표 권장값 | 실전 팁 및 비결 |
| :--- | :--- | :--- | :--- |
| **매수 이격률 (N%)** | 현재가가 VWAP 대비 몇 % 이하로 벌어졌을 때 지정가 매수 진입할지 설정합니다. | - 지수 ETF: **`0.3% ~ 0.8%`**<br>- 우량주: **`1.0% ~ 2.0%`**<br>- 변동성주: **`1.5% ~ 3.0%`** | 변동성이 큰 개별 종목일수록 일시적인 이격이 크게 발생하므로 N% 값을 다소 넉넉히 주어 저가 매수를 노려야 안전합니다. |
| **매도 이격률 (M%)** | 보유 중 현재가가 VWAP 대비 몇 % 이상 올랐을 때 지정가 매도(익절)할지 설정합니다. | **`0.8% ~ 2.5%`** | 현재가가 VWAP 라인을 다시 상향 돌파할 때도 자동 익절 청산되므로, M%는 목표 기대 수익률 한계치로 설정합니다. |
| **손절 비율 (X%)** | 보유 주식의 평균 매수 단가 대비 하락 시 즉시 시장가 청산할 손실 한도%입니다. | **`1.0% ~ 2.0%`** (통상 **`1.5%`**) | 봇의 지정가 체결 속도와 슬리피지를 고려하여 `-1.5%` 선에서 칼같이 손절을 차단하는 것이 정석입니다. |
| **가상 원금 (원/달러)** | 가상 자동매매(`VIRTUAL_1~3`) 구동 시 각 가상 봇의 개별 시작 원금입니다. | 마켓 구분에 따른 통화 기준<br>- 미국: **`달러 ($)`**<br>- 한국: **`원 (₩)`** | 가상 매매 성과 측정(ROI 등) 및 가용 현금의 기준이 되며, 대시보드의 '가상 원금' 입력 필드에서 설정합니다. |
| **기준 자본금 (원/달러)** | 실제 자동매매(`REAL`) 구동 시 성과 요약 및 일 최대 손실 한도 계산의 분모가 되는 자본금입니다. | 마켓 구분에 따른 통화 기준<br>- 미국: **`달러 ($)`**<br>- 한국: **`원 (₩)`** | 실제 예수금 규모와 별도로 투자자가 임의로 지정하여 봇이 사용할 기준 자금 규모를 명시합니다. |
| **투자 비중 (K%)** | 봇이 보유한 가용 예수금 중 1회 거래당 진입할 투자 자금의 비율(%)입니다. | **`5.0% ~ 20.0%`** (통상 **`10.0%`**) | 미수 방지 및 자금 분할 배분을 위해 10% 안팎으로 쪼개어 여러 가상 봇 혹은 실제 봇에 나누어 집행하는 것이 안정적입니다. |
| **일 최대 손실한도 (%)** | 당일 기준 자산(오전/장시작 시점) 대비 총자산 평가 손실이 이 비율 이상 시 강제 셧다운합니다. | **`3.0% ~ 5.0%`** | 봇의 오작동 및 갑작스러운 시장 폭락 시 시스템이 자동으로 미체결을 취소하고 보유 주식을 전량 매도한 뒤 봇을 정지시키는 최후의 보루 장치입니다. |
| **VWAP 리셋 시각** | 실시간 VWAP 계산의 누적 합계를 0으로 초기화하고 새로 시작하는 기준 시각입니다. | - 미국 주식: **`22:30`** (서머타임 해제 시 **`23:30`**)<br>- 한국 주식: **`09:00`** | 미국 본장 개장 시점이나 한국 장 개장 시점으로 맞춰주셔야 당일의 거래량 가중치 계산이 통계적으로 유효합니다. |
| **거래 시작 시각 (대기 설정)** | 리셋 직후부터 지정한 시각까지 매매 주문을 제출하지 않고 대기하도록 딜레이를 주는 시각입니다. | 개장 30분~1시간 뒤인 **`23:00`** 또는 **`23:30`** | 개장 직후 30분 동안 발생하는 초단기 변동성(Opening Drive)에 의한 **속임수(거짓 이격 돌파)** 진입을 걸러내기 위해, 대기 시간을 주어 시장이 안정화된 뒤부터 봇이 작동하게 유도하는 고급 기법입니다. (미입력 시 즉시 작동) |

## 4. 보조 지표 및 Dev Band 작동 방식

### 📊 RSI 및 ADX 필터 (사용자 직접 지정 방식)
RSI와 ADX 보조 지표를 활용하여 추세가 너무 강한 급락장에서 매수했다가 물리는 리스크를 2차 차단합니다.
* **ADX 필터 (대표값: 25~30)**: ADX 수치가 25~30 이상으로 매우 높다면 강력한 원웨이(One-way) 추세장이 진행 중임을 의미합니다. 이때는 이격이 발생해도 진입을 자동 보류(WAIT)하여 급락장에 칼날을 잡는 현상을 방지합니다.
* **RSI 필터 (대표값: 30~35)**: RSI가 30~35 이하로 충분히 과매도 권역에 도달했을 때에만 지정가 매수를 제출하도록 제한하여 진입 단가의 안전성을 높입니다.

### 📐 Dev Band (표준편차 밴드 자동 연산 방식)
* 사용자가 대시보드에서 `Dev Band` 체크박스를 체크하면, 고정 퍼센트 이격률(`N%`, `M%`)을 적용하는 대신 **볼린저 밴드와 유사한 통계적 변동성 적응 밴드**가 활성화됩니다.
* 봇은 실시간 캔들의 거래량 가중 표준편차의 **2.0배수(2.0 시그마)** 밴드 폭을 매수/매도 타겟으로 스스로 연산합니다. 
* 통계적으로 가격이 2시그마 밴드를 일시적으로 벗어났다가 회귀할 확률이 약 **95.4%**에 달하므로, 변동성이 수시로 변하는 개별 잡주 매매 시 이 값을 활성화하면 가격 변동성에 맞추어 타겟가가 자동 조절되어 매우 유리합니다. (사용자는 수치를 입력할 필요 없이 체크박스만 켜시면 됩니다.)

## 5. 금융 안전장치 및 미수 방지 정책

1. **순수 현금(Cash) 기반 예산 검증**: 봇은 매수 주문 전 예수금 상태를 이중 점검하여 주문 예정 총액이 보유 현금을 초과할 경우 진입을 제한하거나 수량을 자동 축소해 미수금 발생 위험을 방지합니다.
2. **현금 100% 매수 주문 보장**: 토스증권 API 호출 시 신용융자가 불가능한 **현금 주문(CASH)** 형태를 강제함으로써 미수/신용 거래 위험을 원천 무력화합니다.
3. **당일 손실 한도 초과 시 시스템 강제 중지**: 당일 평가 자산이 설정된 일 최대 손실한도(예: -5%)를 돌파하면 봇 스스로 기동 중인 모든 지정가 주문을 즉시 취소하고 보유 주식을 전량 시장가로 매도(Panic Sell) 처리한 뒤 봇 작동을 정지(STOP)시킵니다.

## 6. 토스 실거래 듀얼 자산/수익률(ROI) 표기 가이드

실제 거래 대시보드에서는 사용자의 혼선을 줄이고 알고리즘 봇의 독립적인 성과 분석을 위해 **듀얼 모드 표기(Dual Display)**를 적용하고 있습니다.

1. **🏦 TOSS 계좌 전체**:
   * 토스증권 API를 통해 계좌 내 보유한 모든 예수금과 주식 평가금액을 실시간 합산하여 계좌 전체의 총자산을 요약합니다.
   * 계좌 전체 ROI는 `(계좌 총자산 - 설정 자본금) / 설정 자본금 * 100`으로 계산됩니다. 이는 사용자가 계좌 내에서 수동으로 거래하는 종목들의 평가금이 모두 포함된 값입니다.
2. **🤖 VWAP 봇 성과**:
   * 알고리즘 봇이 거래한 순수 내역과 봇이 보유 중인 타겟 종목만을 추적하여 격리 분석한 성과를 제공합니다.
   * **누적 실현 손익 (Realized PnL)**: `vwap_trades_real.json` 파일에 저장된 봇의 과거 체결 완료(매도 청산) 내역의 실현 손익 합산.
   * **현재 평가 손익 (Unrealized PnL)**: 봇이 현재 매매 대상 종목(`real_ticker`, 예: AAPL)을 보유 중일 때, 현재가와 매수 평단가의 이격으로 발생하는 평가 손익.
   * **봇 전용 누적 손익**: `봇 누적 실현 손익 + 봇 현재 평가 손익`
   * **봇 전용 ROI (%)**: `봇 전용 누적 손익 / 기준 자본금 * 100`

이러한 분리 기법을 통해, 단일 계좌 내에서 수동 투자를 병행하더라도 봇의 전략 성과(수익률, 승률)를 완벽하게 격리하여 모니터링할 수 있습니다.
