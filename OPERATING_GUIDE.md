# 🛠️ Butler AI 운영 및 환경별 실행 가이드

본 문서는 PC(SSH), S9 직접 조작 등 다양한 환경에서 **Butler AI v3**를 관리하고 유지보수하는 방법을 설명합니다.

---

## 1. PM2 시스템의 이해 (핵심 원리)
Butler AI는 **PM2**라는 프로세스 관리자 위에서 동작합니다. PM2는 '데몬(Daemon)' 방식으로 동작하기 때문에 다음과 같은 특성을 가집니다.
- **세션 독립성**: SSH 접속을 끊거나 PC를 꺼도 S9 내부에서 서비스는 계속 실행됩니다.
- **공유 관리**: 어느 터미널(SSH 세션 1, 세션 2, S9 직접 조작 등)에서나 `pm2 list`를 입력하면 동일한 서비스 상태를 확인하고 제어할 수 있습니다.

---

## 2. 상황별 실행 및 관리 방법

### A. PC에서 SSH로 접속했을 때 (평상시 관리)
가장 권장되는 관리 방식입니다. 큰 화면과 키보드로 상세 로그를 분석하기 좋습니다.
1. **상태 확인**: `pm2 list`
2. **로그 모니터링**: `pm2 logs butler --lines 100` (최근 100줄 출력 및 실시간 추적)
3. **리소스 감시**: `pm2 monit` (대시보드 형태의 UI)

### B. S9 폰에서 직접 Termux 앱을 열었을 때 (모바일 관리)
외부에서 급하게 상태를 확인하거나 PC가 없을 때 사용합니다.
1. Termux 앱 실행.
2. 그냥 `pm2 list` 입력하여 가동 여부 확인.
3. 문제가 있다면 `pm2 restart all`로 전체 재시작.

### C. 시스템을 재부팅했을 때 (S9 재시작 후)
S9이 꺼졌다가 다시 켜지면 PM2가 자동으로 서비스를 살려냅니다. (이미 `pm2 save` 처리가 되어 있음)
1. Termux 앱 실행.
2. `pm2 list`를 쳐서 서비스가 자동으로 올라왔는지 확인.
3. 만약 안 올라왔다면 `pm2 resurrect` 입력.

## 3. 새로운 작업 시 필수 단계 (가상환경)

단순한 확인(`list`, `logs`) 외에, 새로운 파일을 실행하거나 코드를 수정하여 반영할 때는 반드시 **Python 가상환경(venv)**을 활성화해야 합니다.

```bash
# 1. 프로젝트 폴더 이동
cd ~/dev_pjt/my_butler/

# 2. 가상환경 활성화 (필수)
source venv/bin/activate

# (이제 필요한 작업을 수행하세요)
# 예: 새로운 스크립트 실행 등
```

---

## 4. 안정적인 운영을 위한 체크리스트

1. **Wake Lock 활성화 (필수)**:
   - S9 상단 알림창 -> Termux 알림 -> `Acquire Wake Lock` 클릭.
   - 안드로이드 시스템이 Termux를 강제 종료하는 것을 방지합니다.
2. **배터리 최적화 제외**:
   - S9 설정 -> 애플리케이션 -> Termux -> 배터리 -> `제한 없음` 또는 `최적화 안 함`으로 설정 권장.
3. **PM2 상태 저장**:
   - 서비스를 새로 추가하거나 설정을 변경했다면 반드시 `pm2 save`를 입력하여 현재 상태를 저장하세요.

---

## 5. 긴급 대응 명령어 요약

| 상황 | 명령어 |
|---|---|
| 전체 서비스 일시 정지 | `pm2 stop all` |
| 전체 서비스 재시작 | `pm2 restart all` |
| 특정 서비스만 재시작 | `pm2 restart butler` |
| 로그 파일 비우기 | `pm2 flush` |
| PM2 자체 종료 | `pm2 kill` |
